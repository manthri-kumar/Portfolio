Project Overview

Why Daily Cart?

Problem Statement

Traditional grocery shopping platforms often suffer from poor usability, cluttered interfaces, and inconsistent shopping experiences that frustrate users and reduce conversion rates.

Solution

Daily Cart provides a clean, responsive, and scalable shopping experience with secure JWT authentication and real-time cart updates — making online grocery shopping effortless.

Development Journey

From concept to deployment — the full engineering lifecycle.

01

Planning

Defined project scope, user stories, and database schema. Wireframed core flows — browse, cart, checkout.

02

UI Design

Designed responsive layouts with a dark-themed, product-first visual language using CSS custom properties.

03

Frontend Development

Built all pages with vanilla HTML, CSS, and JavaScript — product cards, filters, and dynamic cart UI.

04

Backend Integration

Developed Node.js REST API with Express, connected MySQL database, and implemented JWT auth middleware.

Testing

Manual end-to-end testing of user flows, API endpoints, authentication edge cases, and responsive breakpoints.

Engineering

Challenges & Learnings

Real problems encountered and how they were solved.

Challenge

Managing Cart State

Maintaining consistent cart state across page navigations without a framework was complex and error-prone.

Learning

Implemented efficient client-side state management using LocalStorage with a centralized cart module, enabling persistent state across sessions.

Challenge

Authentication Flow

Building secure stateless authentication from scratch without third-party libraries required deep understanding of token-based auth.

Learning

Gained deep expertise in JWT implementation — signing, verifying, and invalidating tokens — plus secure storage patterns on the client.

Challenge

Database Communication

Designing efficient SQL queries for product filtering, user lookups, and order creation while avoiding N+1 problems.

Learning

Learned backend integration best practices — parameterized queries, connection pooling, and proper API response shaping with Node.js.
